# VoLTE Mutation Fuzzer

> [!info] 한 줄 요약
> VoLTE/IMS 단말의 SIP 스택 견고성을 검증하기 위한 **터미널 기반 SIP 뮤테이션 퍼저**.

## 목적

VoLTE/IMS 환경에서 동작하는 모바일 단말(UE)의 SIP 스택에 **비정상 SIP 패킷**을 전송하여 크래시, ANR, 재부팅 등 이상 동작을 탐지하는 **블랙박스 퍼징 프레임워크**이다.

> [!important] 핵심 가치
> - IANA 기준 **14개 요청 메서드 + 75개 응답 코드** 전수 커버
> - **3계층 뮤테이션** (모델 / 와이어 / 바이트)으로 다양한 결함 탐색
> - 시드 기반 **재현 가능한** 뮤테이션
> - 소프트폰(Baresip) 및 실제 UE 직접 전송 지원

## 아키텍처

### 파이프라인 개요

```
┌────────────┐    ┌────────────┐    ┌──────────────────┐    ┌────────────┐
│  Generator │ →  │  Mutator   │ →  │ Sender / Reactor │ →  │   Oracle   │
│            │    │            │    │                  │    │  (Phase 5) │
│ 정상 SIP   │    │ 뮤테이션    │    │ 소프트폰 / 실UE   │    │ 크래시 탐지  │
│ 패킷 생성   │    │ 적용       │    │ 전송 + 응답 수집   │    │            │
└────────────┘    └────────────┘    └──────────────────┘    └────────────┘
```

> [!note] 현재 구현 상태
> Phase 1~4 구현 완료. Phase 5(Oracle), Phase 6(Controller)은 미구현.

### 모듈 구조

```
src/volte_mutation_fuzzer/
├── sip/              # SIP 프로토콜 모델 (카탈로그, 공통 타입, 렌더링)
├── generator/        # 정상 베이스라인 패킷 생성
├── mutator/          # 뮤테이션 엔진 (3계층)
├── sender/           # 패킷 전송 및 응답 수집
├── softphone.py      # Baresip 런처
└── packet_docs.py    # 패킷 문서 생성 유틸
```

## 핵심 모듈 상세

### 1. SIP 프로토콜 기반 (`sip/`)

SIP 메시지의 **Pydantic 기반 타입 시스템**.

| 파일 | 역할 |
|------|------|
| `catalog.py` | `SIPCatalog` — 14개 요청 + 75개 응답 중앙 레지스트리 |
| `common.py` | `SIPURI`, `ViaHeader`, `CSeqHeader`, `NameAddress` 등 기본 타입 |
| `requests.py` | 14개 요청 메서드별 스키마 (INVITE, REGISTER, OPTIONS 등) |
| `responses.py` | 75개 응답 코드별 스키마 (1xx ~ 6xx) |
| `render.py` | Pydantic 모델 → SIP 와이어 포맷 변환 |

> [!tip] 지원 메서드
> `ACK` `BYE` `CANCEL` `INFO` `INVITE` `MESSAGE` `NOTIFY` `OPTIONS` `PRACK` `PUBLISH` `REFER` `REGISTER` `SUBSCRIBE` `UPDATE`

### 2. Generator (`generator/`)

**정상(valid) SIP 패킷**을 생성하는 베이스라인 엔진.

- `SIPGenerator`: 설정 기반 SIP 요청/응답 생성 오케스트레이터
- `GeneratorSettings`: 환경변수(`VMF_GENERATOR_*`)로부터 기본값 로드
- `RequestSpec` / `ResponseSpec`: 생성 요구사항 명세
- `DialogContext`: Call-ID, 태그, CSeq 등 다이얼로그 상태

**흐름:**
```
GeneratorSettings.from_env()
  → SIPGenerator(settings)
  → generate_request(RequestSpec) / generate_response(ResponseSpec)
  → SIPRequest / SIPResponse (Pydantic)
  → JSON stdout
```

### 3. Mutator (`mutator/`)

정상 패킷에 **RFC-aware 뮤테이션**을 적용하는 핵심 엔진.

#### 3계층 뮤테이션

| 계층 | 설명 | 예시 |
|------|------|------|
| **model** | Pydantic 필드 값 변경 | `call_id` 교체, `max_forwards` 조작, `cseq.sequence` 변경 |
| **wire** | 헤더 구조 조작 | 헤더 삭제, 중복, 순서 뒤바꿈 |
| **byte** | 원시 바이트 조작 | 비트 플립, CRLF 구분자 변조 |

#### 뮤테이션 전략

- `header_chaos`: 헤더 구조를 무작위로 파괴
- `state_breaker`: 다이얼로그/트랜잭션 상태 위반 유도
- `random_mutations`: 무작위 필드 변경

> [!important] 재현성
> 모든 뮤테이션은 `seed` 값으로 결정적 재현이 가능하다.

**핵심 타입:**
- `EditableSIPMessage`: 안전한 조작을 위한 중간 표현
- `EditablePacketBytes`: 바이트 레벨 조작용 표현
- `MutatedCase`: 원본 패킷 + 적용된 뮤테이션 + 결과 패킷

### 4. Sender/Reactor (`sender/`)

뮤테이션된 패킷을 **전송하고 응답을 수집**하는 모듈.

#### 두 가지 전송 모드

| 모드 | 대상 | 설명 |
|------|------|------|
| `softphone` | 로컬 Baresip | `localhost:5060` 소프트폰 대상 테스트 |
| `real-ue-direct` | 실제 UE | P-CSCF/S-CSCF 경유 실단말 직접 전송 |

- `SIPSenderReactor`: UDP/TCP 전송, Call-ID+CSeq 기반 요청-응답 상관관계 추적
- `RealUEDirectResolver`: MSISDN → UE 연락처 해석 (Docker 컨테이너, PyHSS 연동)
- `DeliveryOutcome`: `provisional` | `success` | `error` | `timeout` | `send_error` | `invalid_response`

## CLI 사용법

### 진입점

```bash
uv run fuzzer <subcommand>
```

### 패킷 생성

```bash
# 요청 생성
uv run fuzzer request OPTIONS
uv run fuzzer request INVITE --override '{"Max-Forwards": 10}'

# 응답 생성
uv run fuzzer response 200 INVITE \
  --context '{"call_id":"c1","local_tag":"tag","local_cseq":7}'
```

### 뮤테이션

```bash
# 직접 생성 + 뮤테이션
uv run fuzzer mutate request OPTIONS --strategy state_breaker --layer model

# 파이프라인 뮤테이션
uv run fuzzer request OPTIONS | \
  uv run fuzzer mutate packet --strategy header_chaos --layer wire
```

### 전송

```bash
# 소프트폰 대상
uv run fuzzer send request OPTIONS \
  --target-host 127.0.0.1 --target-port 5060

# 실 UE 대상
uv run fuzzer send packet \
  --mode real-ue-direct --target-msisdn 222222

# 전체 파이프라인
uv run fuzzer mutate request INVITE --layer byte | \
  uv run fuzzer send packet --target-host 10.20.20.2
```

## 설계 결정

> [!abstract] 핵심 설계 원칙

1. **Pydantic 기반 검증** — 모든 SIP 구조체에 런타임 검증 + JSON 직렬화 적용
2. **3계층 뮤테이션** — 시맨틱(model) / 구조(wire) / 비트(byte) 분리로 유연한 결함 탐색
3. **카탈로그 주도** — 14개 요청 + 75개 응답을 중앙 카탈로그에서 일관 관리
4. **환경변수 기반 설정** — Docker/실험실 배포 지원
5. **소프트폰 우선 전략** — 로컬 Baresip 검증 후 실 UE 전송으로 단계적 확장
6. **동기 I/O** — 단순성 우선, 비동기는 추후 확장 가능
7. **파이프라인 조합** — Unix 철학에 따라 `generate | mutate | send` 파이프 체인 지원
