# 2026-04-19 워크로그

오늘 작업은 **real-ue-direct용 실제 IPsec(native) 모드 구현 → campaign/sender 통합 → 리뷰 기반 하드닝 → 문서 반영 → 최종 회귀 검증** 순서로 진행했다.

핵심 목표는 기존 `null` / `bypass` 평문 우회 모드를 유지하면서,
별도의 `native` 모드를 추가해 **실제 협상된 IMS IPsec/xfrm 경로를 사용하는 송신 경로**를 만들고,
응답도 기존 퍼저와 비슷한 방식으로 확인 가능하게 만드는 것이었다.

이번 세션은 실험 스파이크보다 **구현 완결성과 재현 가능성**에 초점을 맞췄다.

---

## 1. 작업 범위와 기준

이번 라운드의 기준은 다음과 같았다.

- 우선순위 기준 경로: `real-ue-direct`
- 소프트폰 모드: 기본 범위 밖
- 초기 REGISTER 평문: 이번 범위에서 허용
- MT 경로에서도 응답 확인 가능해야 함
- malformed payload를 다룰 수 있어야 함

구현 목표를 코드 수준으로 풀면 아래와 같았다.

1. `--ipsec-mode native` 추가
2. CLI / 계약 모델 / sender / campaign 전체에 native 모드 전달
3. 실제 xfrm state를 기준으로 UE protected port / P-CSCF protected port를 해석
4. raw injector + observer 조합으로 native 송신 및 응답 관측
5. MT-template path와 일반 real-UE path 모두 native 의미를 일관되게 처리
6. replay command, evidence, teardown까지 native와 맞게 정리

## 2. 구현한 큰 변경

### 2-1. CLI / 계약 모델에 `native` 모드 추가

변경 파일:

- `src/volte_mutation_fuzzer/campaign/contracts.py`
- `src/volte_mutation_fuzzer/campaign/cli.py`
- `src/volte_mutation_fuzzer/sender/contracts.py`
- `src/volte_mutation_fuzzer/sender/cli.py`
- `tests/campaign/test_contracts.py`
- `tests/sender/test_cli.py`

핵심 변경:

- `ipsec_mode` 허용값에 `native` 추가
- `ipsec` 별칭을 내부적으로 `native`로 정규화
- `real-ue-direct + native`는 이제 `target_msisdn`을 필수로 요구
- host-only native 호출은 validation 단계에서 명시적으로 거부
- `send packet`, `send request`, `send response` 경로 모두 native 옵션을 동일한 의미로 처리

추가 정리:

- `real-ue-direct + native`는 보호 포트 해석이 필요하므로 `--target-msisdn` 없는 호출을 허용하지 않음
- `softphone`과 non-native real-UE 흐름은 기존 동작을 유지

### 2-2. native IPsec 세션 해석기 추가

변경 파일:

- `src/volte_mutation_fuzzer/sender/real_ue.py`
- `tests/sender/test_real_ue.py`

핵심 변경:

- live xfrm state에서 native IPsec 세션 정보를 해석하는 경로 추가
- `resolve_native_ipsec_session(...)` 구현
- UE protected port와 P-CSCF protected port의 매핑을 구조화
- multi-UE 환경에서 MSISDN이 가리키는 UE IP만 보도록 필터 강화

중요한 후속 보강:

- `resolve_protected_ports(msisdn)`가 더 이상 `msisdn`을 무시하지 않음
- P-CSCF 로그 파싱 시 UE IP까지 함께 읽어 여러 단말이 섞인 로그에서도 해당 UE만 고름
- xfrm fallback도 요청한 UE IP와 P-CSCF IP를 동시에 만족하는 튜플만 사용

운영 메모:

- multi-UE 실험망에서는 `VMF_MSISDN_TO_IP_<msisdn>` 매핑을 잡아둘수록 보호 포트 해석이 더 엄격하고 안정적으로 동작한다

### 2-3. native injector / observer helper 구현

변경 파일:

- `src/volte_mutation_fuzzer/sender/ipsec_native.py` (신규)
- `tests/sender/test_ipsec_native.py` (신규)
- `src/volte_mutation_fuzzer/sender/contracts.py`

핵심 변경:

- raw IPv4/UDP injector helper 구현
- native preflight 구현
- P-CSCF 로그 observer 구현
- `Call-ID`, `Via branch`, 상태코드 기반의 응답 상관관계 추적

응답 관측 방향:

- native에서는 기존 UDP socket `recvfrom()` 방식만으로 응답을 보기 어렵기 때문에
  observer가 P-CSCF 쪽 로그/응답 힌트를 수집해 `SendReceiveResult`에 합성하도록 했다

### 2-4. sender/core에 native 전송 경로 통합

변경 파일:

- `src/volte_mutation_fuzzer/sender/core.py`
- `tests/sender/test_core.py`

핵심 변경:

- `real-ue-direct + ipsec_mode=native` 분기 추가
- generic native path에서도 `target_msisdn` 기준으로 `port_pc`를 다시 해석하도록 수정
- MSISDN 기반 real-UE native 송신이 contact port가 아니라 실제 보호 포트로 preflight/send 되게 보정
- observer event 누적 및 error propagation 정리

의미 있는 수정:

- 이전에는 generic native real-UE send가 SIP contact port를 들고 가다가 native preflight에서 깨질 수 있었음
- 지금은 MT-template path뿐 아니라 일반 native path도 `port_pc` 기준으로 맞춰진다

---

### 2-5. campaign/core에 native MT 경로와 evidence 처리 통합

변경 파일:

- `src/volte_mutation_fuzzer/campaign/core.py`
- `tests/campaign/test_core.py`

핵심 변경:

- `CampaignExecutor`의 base target이 `ipsec_mode`를 유지
- MSISDN-only real-UE 경로에서는 base target에 기본 `5060`을 강제로 들고 시작하지 않도록 수정
- MT-template native target은 `bind_container="pcscf"`, `source_ip=None`, `bind_port=None`로 구성
- null/bypass MT는 계속 `bind_port=mt_local_port` 유지
- native에서는 normal/provisional 응답도 `raw_response`를 남기도록 개선
- non-native에서는 기존처럼 suspicious/crash/stack_failure 시에만 `raw_response` 유지

MT teardown 보강:

- CANCEL이 더 이상 pre-mutation template만 기준으로 생성되지 않음
- wire-layer는 실제 mutate된 wire text 사용
- byte-layer는 실제 전송된 bytes를 decode한 best-effort text를 사용
- 즉, 실제로 보낸 INVITE와 더 가까운 CANCEL이 생성되도록 정리

replay command 보강:

- `--target-host None` 제거
- real-UE 재현 시 가능하면 `--target-msisdn` 기반으로 replay command 생성
- 지원하지 않는 `bind-*` 옵션을 replay 문자열에 넣지 않도록 수정
- `native`인 경우 `--mt-local-port`를 빼도록 정리

---

### 2-6. sender/cli의 real-UE shaping 정리

변경 파일:

- `src/volte_mutation_fuzzer/sender/cli.py`
- `tests/sender/test_cli.py`

핵심 변경:

- `send packet --mode real-ue-direct --ipsec-mode null|bypass`도 `pcscf` 컨테이너 경로를 타게 정리
- `send response`의 softphone/null|bypass 회귀 제거
- MT real-UE는 host-only 호출을 거부하고 `--target-msisdn`을 요구

native MT 특이점:

- `native` MT에서는 Via/Contact를 더 이상 무조건 preserve하지 않음
- 실제 보호 포트에 맞게 sender가 rewrite할 수 있도록 preserve 강제를 풀었다
- 반대로 `null` / `bypass` MT에서는 기존처럼 preserve를 유지한다

이 부분은 마지막 리뷰에서 발견된 실제 리스크였다.
native MT에서 packet source port와 Via/Contact metadata가 어긋날 수 있어서,
native인 경우 auto-preserve를 해제하는 쪽으로 정리했다.

## 3. 리뷰 기반 하드닝

이번 세션은 구현 후 리뷰에서 잡힌 항목을 반복적으로 닫아가며 안정화했다.

주요 수정 사항:

1. `send packet` real-UE null/bypass가 direct host path로 빠지는 회귀 수정
2. MT host-only native 허용 제거
3. `response` softphone 경로에 real-UE용 `bind_container`가 새는 회귀 수정
4. generic native sender가 contact port를 쓰던 문제 수정
5. multi-UE 보호 포트 해석이 다른 단말 포트를 집는 문제 수정
6. MT byte-layer teardown이 실제 전송 payload와 어긋날 수 있는 문제 수정
7. replay command가 실제 CLI로 재실행되지 않는 문제 수정
8. MSISDN-only real-UE campaign이 기본 `5060`을 implicit override로 들고 가는 문제 수정
9. native MT에서 auto-preserve가 sender rewrite와 충돌하는 문제 수정

즉 이번 라운드는 단순히 기능 추가에서 끝나지 않고,
**review-follow-up을 통해 실행 가능한 상태까지 다듬는 데 많은 시간을 썼다.**

## 7. 남은 메모

이번 세션 종료 시점의 운영 메모:

1. `native`는 이제 명시적으로 `target_msisdn`을 요구한다
2. multi-UE 환경에서는 `VMF_MSISDN_TO_IP_<msisdn>` 매핑을 잡는 편이 안정적이다
3. byte-layer MT teardown은 best-effort text 변환을 사용하므로,
   완전히 비UTF-8에 가까운 페이로드에서는 textual CANCEL 생성이 근사치일 수 있다
4. 이번 구현 기준 우선순위는 여전히 `real-ue-direct`이며, 소프트폰 모드는 기본 범위 밖이다
